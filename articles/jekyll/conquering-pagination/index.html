Conquering Pagination

When migrating my site over to Jekyll one requirement was to continue to use the Wordpress theme that I had recently bought for my site. I had a requirement to show only 5 pages in my pagination with the current one being in the middle and styled accordingly. I also wanted to use special arrows for my next and previous pages. Finally I had to have links to the First and Last page, there is quite a bit of logic behind when certain elements are generated which we will get into soon. Below is an example of the pagination requirement.

Page 5 of 33« First←34567→Last »

At first I thought I was going to be able to get this functionality through using Liquid tags that were available in Jekyll however that was not possible. The reason that it could not be completed solely in Liquid is because I needed to do some calculations to figure out what pages to show for instance in the example above the current page is 5. We need to show the next 2 pages and the previous 2 pages in the page numbering. In order to do that you have to do a simple inrement and decrement operation. Liquid will allow you to do basic calculations like this however you have to use the capture tag and that always returns a string. You then cannot use a string an integer in your looping construct, that is when I decided I would have to make this a custom plugin.

The Research

Before I could start writing this plugin I had to find out what Jekyll offered for pagination. I found that it already offered a pagination component but I did not see a way to do anything but list all of your pages in your pager. That looks pretty bad when you have 30+ pages of content. I found that there was only one thing that I would have to do in order to make my custom pagination become a reality. I would need to create a custom Pager class.

Starting the Plugin

The first thing I did when creating this plugin was to create the _plugins\pagination.rb plugin file. The full contents of the plugin are shown below.

module Jekyll
  class Pager

    alias_method :pagination_original_initialize,:initialize
    def initialize(config, page, all_posts, num_pages = nil)

      pagination_original_initialize(config, page, all_posts, num_pages)

      if ( page - 2 ) >= 1 and ( page + 2 ) <= total_pages
        @pager_start = page - 2
        @pager_end = page + 2
      elsif ( page - 2 ) <= 0
        @pager_start = 1
        @pager_end = 5
      elsif ( page + 2 ) >= ( total_pages - 1 )
        if page == total_pages
          @pager_start = page - 4
          @pager_end = page
        elsif page == ( total_pages - 1 )
          @pager_start = page -  3
          @pager_end = total_pages
        end
      end
    end

    alias_method :pagination_original_to_liquid, :to_liquid
    def to_liquid
      pagination_hash_original = pagination_original_to_liquid
      pagination_hash_new = { "pager_start" => @pager_start, "pager_end" => @pager_end }
      return pagination_hash_original.merge(pagination_hash_new)
    end
  end
end

In the Pager class I had to override the initialize method so that I could setup my class instance variables for the start and end of the pagination. You might have noticed I aliased the original method name because I did not want to interrupt the execution of the original Pager class initialize method, I am just adding functionality I am not trying to change the functionality completely. The first thing I do in the initialize method is to call the original initialize method and then I do my calculations for the start and end for the pagination.

I also found that I would have to also override the to_liquid method so that I could add the start and end page numbers into the hash that is used for the pagination template data. With this plugin completed it was time to actually work on my template so I opened up the index.html file in the root of my site and changed it to look like the following.

<h1>My Posts</h1>
<ul class="posts">
{% for post in paginator.posts %}
    <li><span>{{ post.date | date: '%B %d, %Y' }}</span> &raquo; <a href="{{ post.url }}">{{ post.title }}</a></li>  
{% endfor %}
</ul>

<div class="pagination">
    <span class="pagination-pages">Page {{ paginator.page }} of {{ paginator.total_pages }}</span><br/>
    {% if paginator.page > 1 %}
        <a href="/" class="first" title="&laquo; First">&laquo; First</a>
        {% if paginator.page == 2 %}
            <a href="/" >&larr;</a>
        {% else %}
            <a href="/page/{{ paginator.previous_page }}/" >&larr;</a>
        {% endif %}
    {% endif %}


    <!-- pager_start: {{ paginagor.pager_start }}, pager_end: {{ paginator.pager_end }} -->
    {% for count in (paginator.pager_start..paginator.pager_end) %}

      {% if count == paginator.page %}
        <span class="current">{{ count }}</span>
      {% else %}
        <a href="/page/{{ count }}/" class="pagenavi-page" title="{{ count }}">{{ count }}</a>
      {% endif %}
    {% endfor %}

    {% if paginator.page < paginator.total_pages %}
        <a href="/page/{{ paginator.next_page }}/" >&rarr;</a>
        <a href="/page/{{ paginator.total_pages}}/" class="last" title="Last &raquo;">Last &raquo;</a>
    {% endif %}
</div>

Notice that when I loop over and list my posts I am no longer getting them from the site object rather I am pulling them from the paginator object. This is done so that you only display the posts that are assigned to the current page during generation of the site. I have also added some logic to the pagination so that the First and Last page links will only show if you are not currently on one of those pages. Also notice that the looping of the pages is using attribute values from the paginator object for the start and end which were caluclated in the plugin.

Now that we have the majority of the code in place there is still one thing we have to do, we need to tell Jekyll that we wish to use pagination and tell it how many we want to show per page. I also altered the default way pages work by taking setting a paginate_path configuration option so that the pagination would resemble that of what I had with Wordpress. I don't want to lose traffic due to the wrong pages being indexed in Google, so I have tried to follow the same url structure that I had previously. Add the following to your _config.yml file.

pagination: true
paginate: 10
paginate_path: /page/:num

With all of this in place I was able to generate the site and found that the custom pagination was working exactly as expected.
