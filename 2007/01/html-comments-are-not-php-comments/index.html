I am not sure how many times I have run into this. I am not sure if the original developer thinks that the HTML comment will halt the PHP from processing or not. However when you create an HTML comment any PHP that is inside of that comment will still execute and increase the load time for the page. Take this example for instance.<!--more-->[php]<!-- <?=(rand(1,2)%2)?'Hello':'Bye';?> -->[/php]depending on how the rand evaluates the output would be either of the following.[html]<!-- Hello --><!-- Bye -->[/html]As you can see commenting out the HTML will not halt the PHP from executing. If you have a lot of code which is commented out with HTML comments, you are also going to want to comment out the PHP.
